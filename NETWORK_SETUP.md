# Настройка сетевого доступа для Muzz Mobile

## Обзор

Приложение Muzz Mobile настроено для работы с локальным сервером на порту 5000. Конфигурация автоматически адаптируется в зависимости от платформы:

## Настройки URL по платформам

### Desktop (macOS, Windows, Linux)
- **URL**: `http://localhost:5000`
- **Описание**: Прямое подключение к локальному серверу

### Android (Эмулятор)
- **URL**: `http://10.0.2.2:5000`  
- **Описание**: Android эмулятор использует 10.0.2.2 для доступа к host machine

### iOS (Симулятор)
- **URL**: `http://localhost:5000`
- **Описание**: iOS симулятор может напрямую обращаться к localhost

### Web
- **URL**: `http://localhost:5000`
- **Описание**: Web версия работает через localhost

## Права доступа

### macOS
Добавлены следующие entitlements:
- `com.apple.security.network.client`: Разрешает исходящие сетевые соединения
- `com.apple.security.network.server`: Разрешает входящие соединения (если нужно)
- `NSAllowsLocalNetworking`: Разрешает соединения с локальной сетью
- `NSAllowsArbitraryLoads`: Разрешает HTTP соединения (не только HTTPS)

### iOS
Добавлены настройки App Transport Security:
- `NSAllowsLocalNetworking`: true
- `NSAllowsArbitraryLoads`: true

## Запуск локального сервера

Убедитесь, что ваш API сервер запущен на порту 5000:

```bash
# Пример команды для запуска сервера
python app.py  # или ваша команда для запуска сервера
# Сервер должен быть доступен по адресу http://localhost:5000
```

## Отладка соединений

При возникновении проблем с сетью:

1. Убедитесь, что сервер запущен и доступен по указанному адресу
2. Проверьте, что firewall не блокирует соединения
3. Для macOS убедитесь, что приложение имеет права доступа к сети
4. Проверьте консоль приложения на наличие сообщений о конфигурации базового URL

## Настройка для продакшена

Для продакшен версии рекомендуется:
1. Изменить базовый URL на продакшен адрес
2. Использовать HTTPS
3. Удалить `NSAllowsArbitraryLoads` из настроек iOS
4. Настроить proper SSL certificates
